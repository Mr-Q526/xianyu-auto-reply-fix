# 🗺️ 开发路线图 (Roadmap)

本文档记录因安全原因被禁用的功能，以及重新实现这些功能的本地化方案规划。

**核心原则**：所有功能均在本地完成，不依赖任何不可信的第三方服务器，用户数据不出本机/自有服务器。

---

## 📌 功能重新实现计划

### 1. 自动好评功能

**原实现问题**：将用户 Cookie 明文发送到外部 IP `119.29.64.68:8081`，由第三方服务器代为调用闲鱼好评接口。

**本地化方案**：
- [ ] 逆向分析闲鱼好评 API 的调用方式（请求地址、参数、签名算法）
- [ ] 在本地直接调用闲鱼官方好评接口，无需中转服务器
- [ ] 复用现有的 `xianyu_utils.py` 中的签名和加密工具
- [ ] 添加好评延时设置，避免触发风控（建议收货后随机 1-24 小时）
- [ ] 添加好评内容模板，支持随机选择
- [ ] 在 Web 管理界面添加自动好评开关和配置

**优先级**：🔴 高  
**预估工作量**：中等

---

### 2. 通知推送系统（QQ 通知）

**原实现问题**：将消息内容发送到外部 IP `36.111.68.231:3000`，使用不明第三方 QQ 机器人服务。

**本地化方案**：
- [ ] 支持用户自部署的通知服务，由用户自行填写通知 API 地址
- [ ] 对接开源 QQ 机器人框架（如 NapCat、Lagrange 等），用户可在自己的服务器上部署
- [ ] 保留现有的钉钉、飞书、Bark、Telegram 等已有通知渠道（这些不涉及安全问题）
- [ ] 添加 Webhook 通用通知方式，用户可以自定义 HTTP 请求地址

**优先级**：🟡 中  
**预估工作量**：较低

---

### 3. 邮件发送功能

**原实现问题**：将邮件内容（包含验证码）发送到第三方服务器 `dy.zhinianboke.com` 代发。

**本地化方案**：
- [ ] 完善现有的 SMTP 直发功能作为唯一邮件发送方式
- [ ] 在 Web 管理界面添加 SMTP 配置引导（支持 QQ 邮箱、163 邮箱、Gmail 等常见服务商）
- [ ] 添加 SMTP 连接测试功能，配置后可一键测试
- [ ] 提供配置文档说明如何获取各邮箱服务商的 SMTP 授权码

**优先级**：🟡 中  
**预估工作量**：较低

---

### 4. 自动更新模块

**原实现问题**：从不可信的 HTTP 裸 IP（`116.196.116.76`）下载代码并执行，存在严重的远程代码执行风险。

**本地化方案**：
- [ ] 基于 GitHub Releases API 实现安全的版本检查
- [ ] 更新源设置为本项目 GitHub 仓库 `Mr-Q526/xianyu-auto-reply-fix`
- [ ] 使用 HTTPS 通信，校验下载文件的 SHA256 哈希值
- [ ] 仅提示用户有新版本可用，不自动执行更新代码
- [ ] 在 Web 管理界面展示当前版本和最新版本信息

**优先级**：🟢 低  
**预估工作量**：中等

---

### 5. 商品详情自动获取

**原实现问题**：通过第三方 API `selfapi.zhinianboke.com` 获取商品详情，存在数据泄露风险。

**本地化方案**：
- [ ] 利用现有的闲鱼 API 工具直接获取商品详情
- [ ] 使用 Playwright 无头浏览器作为备用获取方式
- [ ] 添加本地缓存机制，减少重复请求
- [ ] 完善商品详情解析逻辑，支持多规格商品

**优先级**：🟡 中  
**预估工作量**：中等

---

### 6. 卡券 API（亦凡 API）替代方案

**原实现问题**：将订单数据和用户账号信息发送到 `78shuk.top` 和 `116.196.116.76`。

**本地化方案**：
- [ ] 设计通用的卡券 API 对接接口规范
- [ ] 支持用户配置自有的卡券发放服务地址
- [ ] 提供本地卡券管理方案（基于现有的数据库卡券管理功能）
- [ ] 添加 API 地址白名单机制，防止误配置到不可信服务器

**优先级**：🟢 低  
**预估工作量**：中等

## 🚀 新功能规划

### 7. 订单销售可视化分析

**功能描述**：提供直观的数据可视化面板，帮助卖家了解销售情况和经营趋势。

**计划实现**：
- [ ] **销售数据仪表盘**
  - 今日/本周/本月销售额、订单数、客单价统计
  - 销售趋势折线图（按日/周/月）
  - 同比/环比增长率分析
- [ ] **商品分析**
  - 热销商品排行榜（按销量、销售额）
  - 商品转化率分析（浏览→咨询→购买漏斗）
  - 滞销商品预警
- [ ] **买家分析**
  - 新客/老客占比
  - 复购率统计
  - 买家地域分布热力图
- [ ] **时段分析**
  - 24 小时订单分布图
  - 最佳发布/回复时段建议
- [ ] **导出功能**
  - 支持导出 Excel/CSV 报表
  - 支持导出可视化图表图片

**技术方案**：
- 前端使用 ECharts / Chart.js 实现图表
- 后端提供数据聚合 API
- 支持自定义时间范围查询

**优先级**：🟡 中  
**预估工作量**：较大

---

### 8. Agent 自动化运营系统

**功能描述**：通过 AI Agent 实现从商品采购到上架发货的全流程自动化，解放人力。

**计划实现**：

#### 8.1 自动采购模块
- [ ] 对接主流虚拟商品平台 API（如发卡平台）
- [ ] 支持配置采购规则（库存阈值、采购数量、价格上限）
- [ ] 库存不足时自动触发采购
- [ ] 采购记录和成本统计

#### 8.2 百度网盘自动上传
- [ ] 对接百度网盘开放 API（OAuth 授权）
- [ ] 采购的文件/资源自动上传到指定网盘目录
- [ ] 自动生成分享链接和提取码
- [ ] 支持按商品/日期自动归档整理

#### 8.3 智能卡券管理
- [ ] 采购完成后自动创建卡券
- [ ] 自动关联商品和卡券
- [ ] 自动设置卡券有效期
- [ ] 库存实时同步和预警

#### 8.4 自动发货规则配置
- [ ] 基于商品关键词自动匹配发货规则
- [ ] 新商品上架时自动创建发货规则
- [ ] 规则模板（支持复制和批量应用）
- [ ] 发货内容自动填充（网盘链接 + 提取码）

#### 8.5 全流程编排引擎
- [ ] 可视化工作流配置界面
- [ ] 触发条件设置（定时/事件驱动）
- [ ] 任务执行日志和异常告警
- [ ] 失败自动重试和人工介入机制

**工作流示例**：
```
库存低于阈值 → 触发采购 → 采购成功 → 上传网盘 → 生成链接 → 创建卡券 → 更新发货规则
```

**技术方案**：
- 基于 asyncio 的任务调度引擎
- 支持 Webhook 触发外部系统
- Agent 决策逻辑可配置（规则引擎 / LLM 辅助）

**优先级**：🔴 高（核心竞争力）  
**预估工作量**：大

## 🔧 安全增强计划

| 项目 | 描述 | 优先级 |
|-----|------|-------|
| 敏感数据加密存储 | Cookie 等凭证在数据库中加密存储，而非明文 | 🔴 高 |
| API 地址白名单 | 对所有外发请求的目标地址进行白名单校验 | 🟡 中 |
| 操作审计日志 | 记录所有涉及敏感数据的操作，便于安全审计 | 🟢 低 |
| HTTPS 强制 | Web 管理界面支持 HTTPS，提供 Let's Encrypt 自动证书配置 | 🟢 低 |
